# Página de Ventas

## Cuándo Usar Este Skill

- Escribir una página de ventas largo formato para un producto digital, curso en línea, programa de coaching o membresía
- Convertir una descripción de producto existente en una página de ventas persuasiva con estructura de conversión clara
- Reconstruir una página de ventas que no está convirtiendo (bajo opt-in, alto bounce rate)
- Crear una página de ventas para un lanzamiento, funnel evergreen u oferta independiente

**NO USES** este skill para landing pages cortas (bajo 500 palabras), páginas opt-in o listados de productos. Esto es para páginas de ventas largo formato que llevan al lector de problema-consciente a listo para comprar.

---

## Prioridad de Elementos de Página de Ventas

Cada página de ventas necesita estos elementos. Construye desde CRÍTICO hacia abajo. Nunca saltes un elemento CRÍTICO.

| Prioridad | Elemento | Impacto | Por Qué Importa |
|---|---|---|---|
| **CRÍTICO** | Headline + Subheadline | Gate de conversión | 80% de visitantes deciden quedarse o salirse basándose solo en el headline |
| **CRÍTICO** | CTA Primario (Botón de Compra) | Driver de ingresos | Un CTA confuso u oculto mata conversiones sin importar calidad de copy |
| **CRÍTICO** | Anclaje de Precio / Valor percibido | Constructor de valor | Sin anclaje, cada precio se siente demasiado alto |
| **CRÍTICO** | Bullets de Beneficios | Constructor de deseo | Transforma características en razones para comprar |
| **ALTO** | Prueba Social / Testimonios | Constructor de confianza | Validación de terceros supera auto-afirmaciones 3:1 |
| **ALTO** | Garantía / Reversión de Riesgo | Destruidor de objeciones | Elimina la barrera #1: "¿Y si no funciona?" |
| **MEDIO** | Sección Manejo de Objeciones | Removedor de fricción | Aborda dudas silenciosas que el lector no expresará |
| **MEDIO** | Urgencia / Escasez | Disparador de acción | Mueve lectores de "después" a "ahora" |
| **MEDIO** | Sección FAQ | Capa de claridad | Atrapa preocupaciones especiales que bloquean el clic final |
| **BAJO** | Acerca del Creador | Suplemento de credibilidad | Solo importa si el lector ya está interesado |
| **BAJO** | Línea P.S. | Nudge final | Atrapa skimmers que van directo al fondo |

## Fase 1: Brief

### Inputs Requeridos

| Input | Qué Preguntar | Default |
|---|---|---|
| **Producto/servicio** | "¿Qué estás vendiendo?" | Sin default — debe ser proporcionado |
| **Precio** | "¿Cuánto cuesta?" | Sin default — debe ser proporcionado |
| **Audiencia** | "¿Para quién es exactamente?" | Sin default — debe ser proporcionado |
| **Resultado principal** | "¿Qué resultado específico obtienen?" | Sin default — debe ser proporcionado |
| **Objeción #1** | "¿Cuál es la razón más común por la que NO compran?" | Sin default — debe ser proporcionado |
| **Prueba social** | "¿Tienes testimonios, números, resultados?" | Sin testimonios — usaremos promesas |
| **Urgencia real** | "¿Hay escasez, fecha límite o razón para actuar ahora?" | Sin urgencia — crearemos contexto |

**PUNTO DE CONTROL: No escribir headline hasta tener producto, audiencia, resultado y objeción principal confirmados.**

## Fase 2: Estructura Completa de Página de Ventas

### Sección 1 — Above The Fold (Lo que ven sin scroll)

```
## Headline Principal

Fórmulas de headline de alta conversión:
- "[Resultado específico] en [tiempo] — sin [obstáculo principal]"
- "Finalmente, [deseo profundo del cliente] — sin [frustración común]"
- "Cómo [persona como ellos] logró [resultado] incluso si [objeción]"
- "El [sistema/método/secreto] que [prueba social numérica]"

**Headline escrito:**
[Headline]

**Subheadline:**
[Expande el headline — clarifica quién es para y qué hace. 1-2 oraciones.]
```

### Sección 2 — Identificación del Problema

```
**Objetivo:** Hacer que el lector sienta que esta página fue escrita solo para ellos.

"¿Alguna vez te has sentido [frustración específica]?"

"[Problema en lenguaje del cliente — no jerga técnica]"

"[Agitar — las consecuencias de NO resolver el problema]"

"No es tu culpa. [Razón externa por la que tienen el problema]."

"Lo que sí es verdad es que [puente hacia la solución]."
```

### Sección 3 — Presentación de la Solución

```
**Presentación del Producto:**
"Presentando [Nombre del Producto] — [descripción de una línea]."

"[Cómo funciona en términos simples — 2-3 oraciones]."

"[Por qué es diferente — el mecanismo único sin hype]."
```

### Sección 4 — Beneficios (Bullets)

```
**Bullets de Beneficios:**
(Formato: Resultado + Detalle + Eliminación de objeción)

✓ [Beneficio 1] — [Detalle específico] (sin necesitar [objeción])
✓ [Beneficio 2] — [Resultado medible] (incluso si [limitación del cliente])
✓ [Beneficio 3] — [Cómo mejora su vida] (sin [sacrificio temido])
✓ [Beneficio 4] — [Victoria rápida que obtienen pronto]
✓ [Beneficio 5] — [Resultado a largo plazo]
```

### Sección 5 — Prueba Social

```
**Testimonios (si disponibles):**

[Formato recomendado:]
"[Resultado específico que obtuvieron]. [Contexto — quiénes son]. [Por qué lo recomiendan]."
— [Nombre, Título/Ciudad, Foto si disponible]

**Si no hay testimonios:**
- Número de clientes/usuarios
- Resultados agregados ("más de X personas lograron Y")
- Menciones o cobertura
- Credenciales del creador relevantes al resultado
```

### Sección 6 — Qué Incluye / Stack de Valor

```
**Stack de Valor:**

"Cuando te unes a [Producto], obtienes:"

[Componente 1]: [Nombre] — [Beneficio] ........... Valor $[X]
[Componente 2]: [Nombre] — [Beneficio] ........... Valor $[X]
[Bonus 1]: [Nombre] — [Beneficio] ................. Valor $[X]
[Bonus 2]: [Nombre] — [Beneficio] ................. Valor $[X]

Valor total: $[suma de arriba]
Tu precio hoy: $[precio real]
```

### Sección 7 — Precio y CTA

```
**Presentación de Precio:**

~~Valor total: $[precio de anclaje]~~
Tu inversión: $[precio real]
[Ahorra $X — [X]% de descuento]

[CTA Button]: "[Quiero acceso ahora / Comprar [Nombre] / Sí, estoy dentro]"

**Garantía:**
"[Garantía de X días / devolución de dinero] — [cómo funciona en 1 oración]."
```

### Sección 8 — Manejo de Objeciones

```
**Objeción 1:** "¿Funciona para [caso específico del cliente]?"
Respuesta: [Aborda directamente con evidence o lógica]

**Objeción 2:** "¿Y si no tengo [tiempo/dinero/experiencia]?"
Respuesta: [Elimina la barrera específica]

**Objeción 3:** "¿Por qué debería confiar en esto?"
Respuesta: [Credenciales + prueba social + garantía]
```

### Sección 9 — FAQ

```
P: [Pregunta frecuente 1 — logística o acceso]
R: [Respuesta directa]

P: [Pregunta frecuente 2 — resultados o timing]
R: [Respuesta con expectativa honesta]

P: [Pregunta frecuente 3 — garantía o cancelación]
R: [Respuesta que elimina riesgo percibido]
```

### Sección 10 — Urgencia y Cierre

```
**Urgencia (si real):**
"[Razón específica por la que actuar ahora — fecha, precio, plazas]."
"Después del [fecha/condición], [consecuencia]."

**Cierre con visión:**
"Imagina [vida del cliente con el resultado]. Eso es lo que [Producto] te da."

**CTA Final:**
"[Botón de compra — mismo texto que arriba]"

**P.S.:**
"[Recordatorio del beneficio principal + urgencia o garantía en 2-3 líneas]"
```

## Anti-Patrones

- **NO** poner CTA solo al final — colocar al menos 3 veces en la página (arriba, medio, fondo)
- **NO** escribir beneficios como características técnicas — "incluye 12 módulos" vs "dominas X en 30 días"
- **NO** headline genérico que podría ser de cualquier producto — debe ser específico a TU audiencia
- **NO** testimonios sin resultado específico — "excelente producto" no vende
- **NO** urgencia falsa — el lector lo detecta y pierde confianza en todo el copy
- **NO** garantía ambigua — "satisfacción garantizada" sin términos claros genera desconfianza
- **NO** página de ventas sin prueba social de ningún tipo — si no tienes clientes, usa resultados propios

## Outputs Esperados

Al completar este skill, el usuario tiene:
1. Página de ventas completa lista para publicar (todas las secciones)
2. 3 opciones de headline para probar A/B
3. Bullets de beneficios escritos en formato de conversión
4. Sección de manejo de objeciones + FAQ

6. CTA copy en 3 posiciones de la página

## Regla de Patrones vs Copia — DAM Vertex

Al crear una landing nueva para DAM Vertex, **no construir solo desde el brief**.

### El flujo correcto

```
Producto nuevo
→ Brief del producto
→ Landings ganadoras existentes (referencia estructural)
→ InSync histórico (referencia de comportamiento)
→ Extracción de patrones ganadores
→ Adaptación al nuevo producto
→ Landing dedicada con identidad propia
```

### Qué es un patrón (extraer esto)

- Orden de secciones que retuvo mejor en landings similares
- Bloques con mayor attention_score o menor abandono posterior
- CTAs con mejor comportamiento según InSync
- Tipo de copy que retuvo (problema-solución / status / regalo / precio)
- Densidad visual y ritmo de sección a sección
- Posición donde aparece: precio visible, prueba social, pago al recibir, entrega en el día
- Secuencia en que se resuelven las objeciones

### Qué NO es un patrón (no copiar esto)

- Colores o paleta de un producto anterior
- Imágenes o producto anterior
- Nombres de variantes
- Testimonios ni copy literal
- Diseño exacto sin adaptar al nuevo producto
- Fondo negro genérico si el producto no lo justifica

### La regla central

**USAR LO GANADOR COMO ESTRUCTURA Y PSICOLOGÍA. NO USARLO COMO PLANTILLA VISUAL LITERAL.**

### Cuando no hay InSync histórico

- Producto nuevo con 0 sesiones: usar los frameworks de esta skill como base
- No inventar datos de comportamiento
- Documentar la decisión basada en framework (no en dato real)
- Revisar InSync a las 2-4 semanas post-lanzamiento para ajustar
