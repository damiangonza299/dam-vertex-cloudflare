# Creador de Bundles

## Principio Fundamental

UN BUNDLE EXCELENTE RESUELVE UN PROBLEMA COMPLETO — NO ES UN DESCUENTO ALEATORIO EN PRODUCTOS RELACIONADOS TIRADOS JUNTOS.

---

## Cuándo Usar

- Usuario quiere aumentar AOV (valor promedio de orden) mediante bundling
- Usuario necesita decidir qué productos agrupar
- Usuario quiere análisis de margen en precios de bundle potenciales
- Usuario necesita copia promocional para oferta de bundle

---

## Tipos de Bundle de Referencia

| Tipo | Lógica | Mejor Para | Descuento Típico |
|---|---|---|---|
| **Starter Kit** | Todo lo que principiante necesita | Adquisición cliente nuevo | 15-20% vs. buying separately |
| **Complete Solution** | Resuelve un problema end-to-end | Conversión mid-funnel | 20-25% |
| **Mix & Match** | Cliente elige X items de categoría | Liquidar inventario | 10-15% |
| **Upgrade Bundle** | Producto principal + premium add-ons | Upselling, AOV increase | 10-15% on add-ons only |
| **Gift Set** | Curado para ocasiones de regalo | Estacional, vacaciones | 15-20% |
| **Subscribe & Save** | Bundle recurrente con auto-ship | Consumibles, retención | 15-25% |
| **Cross-Sell Bundle** | Productos complementarios diferentes | Discovery, basket size | 10-20% |

## Flujo de Trabajo

### Fase 1: Identifica Oportunidades de Bundle

Reúne datos de producto:
- Nombres de producto, precios individuales, costo/margen por unidad
- Qué productos se compran frecuentemente juntos
- Qué productos tienen exceso de inventario
- AOV actual y AOV objetivo
- Segmentos de cliente (nuevo, returning, VIP)

### Fase 2: Diseña el Bundle

Selecciona productos siguiendo estas reglas:
- Incluye 1 anchor product (la razón para comprar el bundle)
- Añade 1-3 productos complementarios que mejoren el anchor
- Asegura cada producto en el bundle tiene sentido lógico juntos
- Incluye al menos un item alto-margen para proteger rentabilidad

---

## Fase 3: Análisis de Precios y Margen

### Framework de Cálculo de Bundle

```
## Análisis de Bundle — [Nombre del Bundle]

**Tipo de Bundle:** [Starter Kit / Complete Solution / Gift Set / etc.]

**Componentes:**
| Producto | Precio Individual | Costo (COGS) | Margen | Incluido en Bundle |
|---|---|---|---|---|
| [Producto 1] | $[X] | $[X] | [X]% | Sí |
| [Producto 2] | $[X] | $[X] | [X]% | Sí |
| [Producto 3] | $[X] | $[X] | [X]% | Sí |

**Precio sumado (sin bundle):** $[X]
**Descuento del bundle:** [X]% = -$[X]
**Precio final del bundle:** $[X]
**Costo total del bundle:** $[X]
**Margen del bundle:** [X]%
**Margen mínimo aceptable:** 30%

¿Bundle viable? [SÍ / NO — ajustar si margen < 30%]
```

---

## Fase 4: Psicología de Precios del Bundle

### Estrategias de Anclaje

| Técnica | Descripción | Ejemplo |
|---|---|---|
| **Valor comparativo** | Mostrar precio sin bundle vs con bundle | "Valor $197 — Tu precio: $127" |
| **Desglose de valor** | Listar valor percibido de cada componente | "Incluye X ($47) + Y ($37) + Z ($27)" |
| **Nombre premium** | Bundle name implica más valor | "Kit Profesional Completo" vs "3 productos" |
| **Edición limitada** | Urgencia por disponibilidad | "Solo 50 kits disponibles" |
| **Economía obvia** | El ahorro supera el precio | "Ahorra $70 — comprando por separado cuesta $197" |

## Fase 5: Copy Promocional del Bundle

### Estructura de Copy de Bundle

```
## Copy del Bundle — [Nombre del Bundle]

**Headline principal:**
"[Nombre del Bundle]: [Beneficio o problema que resuelve completo]"

**Subheadline:**
"[Lo que incluye en una línea + el resultado que obtienen]"

**Bullets de contenido:**
✓ [Producto 1] — [Por qué es valioso / resultado específico]
✓ [Producto 2] — [Por qué es valioso / resultado específico]
✓ [Producto 3] — [Por qué es valioso / resultado específico]

**Valor total:**
"Todo esto vale $[X] si lo compras por separado."

**Precio del bundle:**
"Tu precio hoy: $[X] — ahorras $[X] ([X]%)"

**CTA:**
"[Comprar Bundle / Obtener el Kit Completo / Quiero Todo]"

**Urgencia / escasez (si aplica):**
"[Solo disponible hasta / Oferta por lanzamiento / Stock limitado]"
```

## Fase 6: Placement del Bundle

### Dónde y Cuándo Mostrar el Bundle

| Canal | Momento | Tipo de Bundle |
|---|---|---|
| Página de producto | Antes de agregar al carrito | Cross-sell, Upgrade |
| Checkout | Post add-to-cart, pre-pago | Starter Kit, Add-on |
| Email post-compra | 3-7 días después | Complete Solution |
| Landing ad dedicada | Tráfico frío | Gift Set, Starter Kit |
| Retargeting | Visitantes sin compra | Bundle con descuento temporal |

---

## Anti-Patrones

- **NO** crear bundles con productos que no tienen relación lógica — confunde y no convierte
- **NO** descontar tanto que el margen caiga bajo 25% — no es sostenible
- **NO** incluir el producto de menor margen como anchor — protege rentabilidad
- **NO** crear más de 3-4 opciones de bundle — la paradoja de elección paraliza la compra
- **NO** nombrar el bundle con nombres técnicos — usar nombres que implican el resultado
- **NO** ocultar el ahorro — el valor percibido es el motor del bundle
